    fillFormWithHALData(url, data, id, templateopt, excludeopt)

    Populate a form with the elements contained in the data parameter. The data.parameter name is the name attribute while the data.parameter value represents the value. All parameters are by default assigned as readonly. The action attribute is filled in with the url parameter. If a template is provided, the parameters contained in the template are set editable. Required fields are marked with the required property. If a template property does not have its counterpart it will create a new for that property. NOTE: All buttons in the form are hidden. After executing this method adequate buttons should be shown using $(#button_name).show()
    Parameters:
    Name Type Attributes Description
    url string The url of to be added in the action attribute
    data Object An associative array formatted using HAL format (https://tools.ietf.org/html/draft-kelly-json-hal-07) Each element in the dictionary will create an element in the form.
    id string The id of the form is gonna be populated
    template Object <optional>
    		 a Collection+JSON template (https://github.com/collection-json/spec#23-template) which is utlized to set the properties of the elements in the form, or append missing ones.
    exclude Array <optional>
    		 A list of attributes names from the data parametsr that are not converted in elements.

    user_messages(apiurl)

    Sends an AJAX GET request to retrieve information related to user history. Associated rel attribute: messages ONSUCCESS => a.1) Check the number of messages received (data.collection.items) a.2) Add the previous value to the #messageNumber input element (located in #userHeader section). b.1) Iterate through all messages. b.2) For each message in the history, access the message information by calling the corresponding Message instance (call history_collection_get_item) The url of the message is obtained from the href attribute of the message item. ONERROR => a)Show an *alert* informing the user that the target user history could not be retrieved b)Deselect current user calling #deselectUser.

    Name Type Description
    apiurl string The url of the History instance.

    user_restricted_data(apiurl)

    Sends an AJAX request to retrieve the restricted profile information: User Restricted Profile Associated rel attribute: restricted-data ONSUCCESS => a) Extract all the links relations and its corresponding URLs (href) b) Create a form and fill it with attribute data (semantic descriptors) coming from the request body. The generated form should be embedded into #user_restricted_form. All those tasks are performed by the method #fillFormWithHALData b.1) If "user:edit" relation exists add its href to the form action attribute.
